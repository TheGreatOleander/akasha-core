## Usage

### Ingesting Observations

Add observations with:
- **Type** ‚Äî what kind of event
- **Entity** ‚Äî who/what is involved
- **Location** ‚Äî where it happened (x,y coordinates)
- **State** ‚Äî current condition
- **Notes** ‚Äî additional context

### Viewing Facets

- Toggle lenses on the left to activate different observation facets
- Click facet tabs to see detailed projections
- Combine multiple lenses to see multi-dimensional patterns

### Understanding Voids

Akasha automatically detects:
- **Temporal voids** ‚Äî gaps in time where events should have occurred
- **Spatial voids** ‚Äî locations with unexpected absence of activity
- **Network voids** ‚Äî entities with missing or broken connections
- **State voids** ‚Äî systems that have gone silent

## Architecture

### Core Engine (`AkashaCore`)

The invariant observation substrate. Never changes. All facets operate through this engine.

**Key Methods:**
- `ingest(observation)` ‚Äî Add new observation
- `project(lensStack)` ‚Äî View through selected facets
- `detectVoids()` ‚Äî Find absences and gaps
- `query(filters)` ‚Äî Search observations

### Facet Analyzers

Each facet has dedicated analysis:
- `analyzeTime()` ‚Äî Temporal patterns and cycles
- `analyzeSpace()` ‚Äî Spatial distributions and density
- `analyzeEvent()` ‚Äî Event correlations and clusters
- `analyzeNetwork()` ‚Äî Connection graphs and flow
- `analyzeState()` ‚Äî State transitions and stability
- `analyzeVoid()` ‚Äî Absence detection across all dimensions

## Philosophy Deep Dive

### Why "Observation Without Agency"?

Most systems conflate observation with action:
- They see, then recommend
- They detect, then predict
- They measure, then optimize

Akasha separates these:
- It sees, then shows you
- It detects, then records
- It measures, then maps

**You are the authority.** Akasha is the instrument.

### The Primacy of the Void

Reality is as much about what *isn't* as what *is*:
- The crime that didn't happen
- The connection that broke
- The signal that went silent
- The event that was delayed

Traditional systems ignore absence because it's hard to represent. Akasha makes voids first-class entities.

### Non-Teleological Computing

Akasha has no goal. It doesn't want anything. It doesn't optimize for anything.

It simply watches.

This is rare in software. Most systems are purpose-built. Akasha is purpose-neutral.
